Location

Many of the key classes provided by SOSA and SSN represent entities that can be located in space such as sensors, features of interest, actuators, samples, or activities that can be located via their participating entities, e.g., platforms. These entities will usually be described using models and ontologies defined for application domains, including technical disciplines, social and business contexts.

Standards for spatial properties are discussed in the Spatial Data on the Web Best Practices note [[SDW-BP]]. In particular, GeoSPARQL [[geosparql]] provides a flexible and relatively complete platform for geospatial objects, to support interoperability between geo-datasets. Using GeoSPARQL, geometries can be assigned to entities using the geo:hasGeometry property.
