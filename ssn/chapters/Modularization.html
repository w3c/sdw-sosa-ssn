SSN Ontology Modularization

The SSN Ontology is distributed in several modules. The segmentation allows users to access the knowledge they require, reducing the scope to what is necessary in a given use case. Lower level modules are independent of their higher level modules and logically consistent on their own.

Higher level here is not to be confused with upper level ontologies. Upper level ontologies are general knowledge ontologies that can be directionally imported in many domains.

Core modules

This figure summarizes the modules and dependencies within the core SSN Ontology.

SOSA

The SOSA Module (Sensors, Observations, Samples and Actuations) defines the core classes and properties, and provides textual definitions and other annotations. Some semantic enrichment is provided using Schema.org [[schema-org]] properties, but there is no formal axiomatization. SOSA may be used as-is by an audience who wish to use terminology consistent with the SSN Ontology, thus ensuring interoperability with SSN-based repositories.

SOSA is composed of four lower level modules:

  1. SOSA Common defines classes and properties that are shared across Actuation, Observation and Sampling applications
  2. SOSA Actuation imports SOSA Common and defines classes and properties used in Actuations
  3. SOSA Observation imports SOSA Common and defines classes and properties used in Observations
  4. SOSA Sampling imports SOSA Common and defines classes and properties used in Samplings

Finally, SOSA imports SOSA Actuation , SOSA Observation and SOSA Sampling .

SOSA as the core, does not import any other ontologies, so it is truly independent of modules that add more expressivity and further ontological commitments to the lightweight semantics of SOSA.

SSN

The SSN Module adds axioms that formally indicate how properties relate to classes, how classes relate to each other. SSN may be used in semantic web systems that use inferencing and reasoning, and consistency checking for datasets.

The modularization of SSN mirrors SOSA. SSN is composed of four lower level modules:

  1. SSN Common imports SOSA Common and axiomatizes classes and properties that are shared across Actuation, Observation and Sampling applications
  2. SSN Actuation imports SSN Common and SOSA Actuation, and axiomatizes classes and properties used in Actuations
  3. SSN Observation imports SSN Common and SOSA Observation, and axiomatizes classes and properties used in Observations
  4. SSN Sampling imports SSN Common and SOSA Sampling, and axiomatizes classes and properties used in Samplings

Finally, SSN imports SSN Actuation , SSN Observation and SSN Sampling .

Readers who do not wish to view the SSN axioms may hide them by selecting the button [Show only core SOSA terms] at the top of the Overview of Classes and Properties or [Hide additional SSN axioms] on each individual specification block.

Extension modules

This figure summarizes the extension modules described in this document and their dependencies on the SSN Ontology.

System Capabilities

The System Capabilities module imports SSN Actuation and SSN Observation, and adds classes and properties required for the description of the capabilities of systems, in particular Sensors and Actuators.

Sample Relations

The Sample Relations module imports SOSA Sampling, and adds classes and properties required for the description of networks of samples.

Alignment modules

This figure summarizes alignments of other ontologies and specifications with the SSN Ontology that are described in this document.

OMS

The SOSA-OMS module imports SOSA Observation and SOSA Sampling and describes the alignment of SOSA to [[[OMS]]], adding some additional terms used in OMS that are not otherwise included in SSN. SOSA-OMS may be used by audiences for whom strict conformance to [[OMS]] (also known as ISO 19156:2023) is a requirement

The SSN-OMS module imports SOSA-OMS and SSN Observation and SSN Sampling and describes the alignment of SSN to [[OMS]] including OWL/RDFS axiomatization. SSN-OMS may be used by audiences for whom strict conformance to [[OMS]] (also known as ISO 19156:2023) is a requirement, with reasoning support

PROV

The SOSA-PROV module imports SOSA Common, and describes the alignment of SOSA to [[[prov-o]]]. SOSA-PROV may be used to integrate data designed using SSN into provenance systems.

OBOE

The SOSA-OBOE module imports SOSA Observation, and describes the alignment of SOSA to [[[OBOE]]]. SOSA-OBOE may be used to integrate observation data structured using OBOE with data using SOSA.

SAREF

The SOSA-SAREF module imports SOSA, and describes the alignment of SOSA to [[[SAREF]]]. SOSA-SAREF may be used to integrate data designed using SSN with data designed using SAREF.

DUL

The SSN-DUL module imports SSN, and adds axioms that position SSN within Dolce-UltraLite. SSN-DUL may be used to integrate data designed using SSN into systems otherwise built in the Dolce-UltraLite framework.
